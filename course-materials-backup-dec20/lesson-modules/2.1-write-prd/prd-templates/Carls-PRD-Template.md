# Carl's PRD Template

## 1. Problem Alignment
### The Problem
*What is the problem we are solving? Why now?*

### Goals & Success Metrics
*How will we know we've solved the problem? (KPIs, North Star)*

### User Personas
*Who are we building for?*

### User Stories
*As a [persona], I want to [action], so that [value].*

## 2. Solution Alignment
### Proposed Solution
*High-level overview of the solution.*

### Key Features
*Detailed list of features.*

### User Flow
*How does the user navigate the feature?*

### Design Requirements
*Look and feel, UI components.*

## 3. Development Planning
### Technical Requirements
*Frontend, Backend, APIs, Infrastructure.*

### Implementation Phases
*Phase 1 (MVP), Phase 2, etc.*

### Risks & Mitigations
*What could go wrong and how will we handle it?*

### Open Questions
*Things still to be decided.*
